# 多模态多领域假新闻检测方法路线

日期：2026-05-23

状态更新（2026-05-28）：本文档用于领域路线和 related work 沉淀，非当前执行清单。Round 6/7 当前作用域验证已闭环；新的待执行规划入口为 Round 7 `Risk-Aware Final-Boundary Learning`，见 `新创新方案_Round7PANDA_RiskAwareFinalBoundary.md`、`round7_candidate_registry.md`、`round7_gate0_manifest_TEMPLATE.json` 和 `todo.md`。

## 文档用途

本文档用于沉淀 PANDA 复现完成后整理出的领域方法路线，服务后续创新方案设计、related work 写作和实验 baseline 选择。

本项目聚焦中文多模态多领域假新闻检测。核心任务是输入新闻文本、图片和领域/domain，输出真假新闻二分类概率或标签。当前复现底座是 PANDA：Prototype-driven Asymmetric Neighbor-Domain Adaptation，重点问题是多领域知识迁移中的 negative transfer。

## 当前项目锚点

项目已完成 PANDA 在 Weibo-21 与 Weibo 上的 paper-aligned 三 seed 复现：

| Dataset | Macro-F1 | Accuracy | AUC |
| --- | ---: | ---: | ---: |
| Weibo-21 | 0.9474 ± 0.0073 | 0.9474 ± 0.0073 | 0.9879 ± 0.0014 |
| Weibo | 0.9415 ± 0.0034 | 0.9415 ± 0.0034 | 0.9866 ± 0.0011 |

关键观察：

- AUC 基本对齐 PANDA 论文 reported 结果。
- Macro-F1 和 Accuracy 比 PANDA reported 低约 1 个百分点。
- MMDFND / DAMMFND reproduced baselines 已完成两数据集三 seed；正式主表应使用 reproduced baseline，而不是 paper reported 单点。
- PANDA reproduced 当前仍是最强 baseline：Weibo-21 Macro-F1 0.9474 ± 0.0073，Weibo Macro-F1 0.9415 ± 0.0034。
- 后续创新实验应统一使用本项目三 seed mean ± std 作为 PANDA reproduced baseline，而不是直接使用论文单点 reported 值。
- 2026-05-24 gate 后，Conflict-aware Stable Neighbor-Domain Adaptation、clean Reliability-aware Disagreement selector 与 uncertainty-aware stable-source selection 均未通过稳定性要求；这些结果作为证据池，不作为主线终点。
- 2026-05-25 至 2026-05-26，R3-PANDA Regret Router、历史 R4 non-self source-domain intervention consistency、DCA/source-view sensitivity、feature-aware PAD、domain-conditioned expert gate audit、evidential/reliability extension 均已完成对应 gate；2026-05-27 复审后必须按作用域读取：P1-B 是 `Blocked/current checkpoint unsupported`，D2/D3 只关闭当前 probe/offline/frozen 变体。
- Round 2 P2-A/P2-B/P2-C/P2-D 已完成并无 `Primary-Candidate`；P2-B `final+aux logits` residual adapter 曾是 `Feasible-B` 弱信号，但 Round 3 branch-boundary residual 强对照复核后对当前 D3 offline residual No-Go。
- 当前方法主线进入 Round 4 first-principle boundary rebuild 候选池，方案文档为 `新创新方案_Round4PANDA_FirstPrincipleBoundary.md`。
- CLIP-only 不强，不能主打 cross-modal conflict causality；branch disagreement、fusion uncertainty 与 confidence-uncertainty 是当前更可靠的证据来源。
- Gate 已限定为 train/val 设计与验证、test 最终确认，避免 test-set 过拟合和 full-conflict 监督泄漏。
- Round 4 方法路线必须证明 class-conditional geometry、low-rank error subspace、memory diagnostic 或 domain shortcut audit 中至少一条能解释并改变 final-boundary failure，且不能被 calibration-only、ordinary stacking、weighted average、deterministic、duplicate、random、shuffled、label-shuffle 或 parameter-matched controls 解释。
- 2026-05-25 新增总控协议后，任何方案过最小 gate 也不直接封主线；必须继续完成候选池中其余方案的最小可行性验证，最后再从可行方案中确定最终主线。
- Gate 分为 pre-method signal gate 和 offline selector behavior gate；现有结果只支持 Weibo-21 有弱机制苗头，Weibo 作为边界条件处理。
- Gate 执行顺序固定为 pre-method signal gate -> minimal/offline selector behavior gate -> test confirmatory -> post-implementation mechanism gate。
- 正式弱域由 train/val baseline 选择，test 不参与弱域定义。
- High-confidence error 分析需要 overconfidence-only 负对照。

## 方法路线总览

### 路线一：文本、传播与事件泛化

早期假新闻检测主要依赖文本内容、用户特征、传播结构和事件信息。多模态方法出现前，这类方法重点解决文本语义、事件泛化和社交上下文建模。

代表方法：

- EANN：Event Adversarial Neural Networks for Multi-Modal Fake News Detection。通过事件对抗学习减少事件特异性特征，提高跨事件泛化能力。
- 单模态文本分类器和传播图方法：通常作为弱 baseline 或补充特征来源。

适用价值：

- 可以作为 related work 的历史起点。
- 对本项目的直接增量价值有限，因为 PANDA 已经处在多模态多领域阶段。

### 路线二：基础多模态融合

这一阶段将文本、图片、社交特征或图像视觉特征进行拼接、注意力融合或联合表示学习。

代表方法：

- SpotFake / SpotFake+：使用 BERT 和视觉 backbone 做新闻文本与图像融合。
- MVAE：用多模态变分自编码器联合建模文本、图像和社交特征。

核心思想：

- 文本负责语义和主张。
- 图像负责视觉证据和视觉语境。
- 融合层学习联合表示后进行真假分类。

局限：

- 容易把不同模态简单拼接，缺少对图文一致性和领域差异的显式建模。
- 面对跨领域数据时，可能学习到领域偏置而非可迁移事实线索。

### 路线三：图文一致性与跨模态歧义

假新闻常见问题不是单模态内容本身虚假，而是图文错配、语境错配或同一图像被挪用。因此这一类方法显式建模图文一致性、相似度或歧义。

代表方法：

- SAFE：Similarity-Aware Multi-modal Fake News Detection，显式使用图文相似度辅助判别。
- CAFE：Cross-modal Ambiguity Learning for Multimodal Fake News Detection，从跨模态歧义角度学习图文不一致信号。

可迁移模块：

- 图文一致性分数。
- 图文冲突/歧义表征。
- 将图文一致性作为辅助 view 输入 PANDA 的 DMPG、PAD 或 GNS。

对本项目的启发：

- PANDA 重点处理领域负迁移，但对图文错配的显式刻画不强。
- 本项目 gate 显示 CLIP-only 图文冲突不强，因此 SAFE/CAFE 风格信号只能作为辅助相关工作和消融项，不能作为主因果证据。

### 路线四：视觉语言预训练、CLIP 与 Prompt

随着视觉语言预训练模型成熟，假新闻检测开始更多依赖 CLIP、ViT、BLIP、MAE、BERT/RoBERTa、prompt tuning 等预训练能力。

代表方向：

- 使用 CLIP 或 Chinese-CLIP 获得图文对齐表示。
- 使用 MAE/ViT 提取图像 token。
- 使用 BERT/RoBERTa 提取文本 token。
- 使用 prompt 或 domain prompt 进行任务适配。

与 PANDA 的关系：

- PANDA 已经使用 RoBERTa、MAE 和 CN-CLIP。
- PANDA 的 DMPG 实质上是 domain-aware modal prompts。
- 后续创新不应简单替换 backbone，因为这会增加成本并模糊贡献边界。

推荐用法：

- 保持 PANDA backbone 不变。
- 优先改进 prompt/prototype/neighbor selection/calibration。
- 若要引入新预训练模型，应作为附加实验，而不是主贡献。

### 路线五：多领域 MoE、Domain Gate 与 Memory

多领域假新闻检测的核心是不同领域存在不同语言风格、视觉模式、事实背景和谣言套路。单一模型容易被领域偏置影响，因此出现 domain gate、MoE、memory bank 等路线。

代表方法：

- MDFEND：Multi-domain Fake News Detection，使用 domain gate 和多专家结构。
- M3FEND：Memory-Guided Multi-View Multi-Domain Fake News Detection，使用语义、情感、风格多视图和 domain memory bank。
- FuDFEND：使用 fuzzy domain label 缓解硬领域标签不足。
- MMDFND：Multi-modal Multi-Domain Fake News Detection，同时处理多模态和多领域。
- DAMMFND：Domain-Aware Multimodal Multi-view Fake News Detection，进一步增强多模态多视图领域建模。

核心模块：

| 模块 | 作用 | 对 PANDA 的启发 |
| --- | --- | --- |
| Domain gate | 根据领域选择专家或融合权重 | 可让 PANDA 旧专家门控显式 domain-conditioned |
| MoE experts | 学习领域共享与领域特定特征 | 可分析 PANDA 是否仍有 domain 0 gate 复用问题 |
| Memory bank | 存储领域原型、视图原型或历史模式 | 可稳定 PANDA 的 domain prototypes |
| Fuzzy domain label | 一条新闻可属于多个软领域 | 可缓解 PANDA 硬 category 对混合新闻的限制 |
| Multi-view features | 语义、情感、风格、视觉、图文一致性 | 可增强 PAD 计算所依赖的领域表征 |

对本项目的价值：

- 这条线是 PANDA 直接上游，必须在 related work 中重点写。
- MMDFND 和 DAMMFND 是后续最需要实跑的 baseline。

### 路线六：负迁移显式缓解与邻域选择

这是 PANDA 所在的核心路线。传统多领域方法容易将所有源域知识一并迁移到目标域，导致无关或冲突领域带来负迁移。PANDA 的贡献是从 blind transfer 转向 wise selection。

PANDA 核心模块：

| 模块 | 全称 | 作用 |
| --- | --- | --- |
| DMPG | Domain-aware Modal Prompt Generation | 为每个领域生成文本、视觉、多模态 prompt |
| PAD | Prototype-based Asymmetric Distance | 用领域原型计算源域到目标域的非对称距离 |
| GNS | Gumbel-based Neighbor Selector | 基于相似度和 Gumbel top-k 动态选择邻域 |
| DCA | Domain-Collaborative Attention | 用目标域 prompt 查询所选邻域 prompt，融合邻域知识 |

本项目复现发现：

- GNS 在 forward 中引入随机噪声，eval 路径仍可能存在输出波动。
- AUC 基本复现，但 F1/Acc 偏低，说明排序能力强而离散分类边界不够稳。
- PANDA 的领域选择机制是创新最自然的切入点。

当前已验证的失败边界：

- clean reliability-aware multimodal disagreement selector 未通过 seed42 gate。
- uncertainty-aware stable-source selection 未通过 seed-recheck。
- R3 regret router 出现 self-domain route collapse。

后续创新应优先围绕：

- R4 non-self source-domain intervention views。
- deterministic eval 作为强对照。
- per-domain negative transfer 诊断。
- counterfactual sensitivity、anchor-view JSD、view flip rate 和 view correct rate。
- ECE、Brier score、repeated eval variance 等可靠性指标。
- 领域级阈值校准只作为辅助分析，不作为主贡献。

### 路线七：知识增强、检索增强与 LLM/MLLM

近年假新闻检测开始引入外部知识、检索、证据生成和多模态大模型。其动机是：只看新闻本身可能不足以判断真假，模型需要外部事实、相似历史案例或背景知识。

代表方向：

- AKA-Fake：知识增强多模态假新闻检测，强调选择有用知识子图。
- FND-LLM：使用 LLM rationale 或多分支模型辅助判别。
- ExDR / RADAR：检索增强或解释驱动检索。
- RAMM：Retrieval-Augmented Multimodal Model，使用 MLLM 与跨实例叙事检索。

注意事项：

- 这类方法很新，部分代码和正式录用状态需要继续核验。
- 直接把 LLM 接入主模型会带来成本、漂移、幻觉和可复现性问题。
- 对 C 会稳投来说，RAG/LLM 不适合作为第一优先主线，除非有足够时间做数据构建、证据质量分析和可复现实验。

推荐用法：

- 作为 related work 和未来方向。
- 若要嫁接到 PANDA，应先做 sidecar，不要替换 PANDA 主干。
- 可以离线生成 entity、claim、rationale 或相似案例 embedding，再作为额外 view。

| P0 | MMDFND / DAMMFND | 两数据集三 seed reproduced baseline 已完成，可直接进入主表 |
| P0 | Round 4 Class-Conditional Boundary Geometry | 当前第一优先候选，验证错误是否来自 class-conditional decision geometry 而不是 source routing |
| P0 | Round 4 Low-rank Error Subspace | 第二优先候选，验证剩余错误是否集中在低维 residual boundary directions |

| P1 | Per-domain error analysis | 能定位负迁移发生在哪些领域，是论文分析关键 |
| P1 | ECE / Brier / repeated eval variance | 支撑可靠性与性能评测角度 |
| P1 | Memory bank / prototype refinement | 可稳定 PAD 和 domain prototypes |
| P1 | Fuzzy domain labels | 适合混合领域新闻，但实现与验证成本更高 |
| P1 | Domain-conditioned expert gate | 旧 checkpoint audit 是 `Blocked/current checkpoint unsupported`，P2-C offline pre-screen 只否定当前 D3 离线变体；未来重开必须新增真实 9-domain modules 并过 parameter-matched/random-route controls，训练期需 D4 |
| P1 | Evidential reliability extension | 只作为 future second-stage add-on；当前不能替代 primary mechanism |
| P2 | Domain-aware threshold calibration | 只能作为可靠性辅助或后处理分析，不宜作为主贡献 |
| P2 | RAG/LLM evidence | 潜力高，但可复现风险与工程成本最高 |

## Baseline 与当前主表选择建议

当前不做旧候选的“正式创新方法主表”。报告/论文主表先放 reproduced baselines；Round 3 只有在三 seed val 通过后才解锁正式新方法主表：

| 类别 | 方法 | 状态 |
| --- | --- | --- |
| 文本/多模态基础 baseline | 可选 text-only、image-only、fusion-only | 作为消融或补充 |
| 多领域 baseline | MDFEND / M3FEND | 视复现成本选择 |
| 多模态多领域 baseline | MMDFND | 已完成 reproduced 三 seed |
| 强多领域多模态 baseline | DAMMFND | 已完成 reproduced 三 seed |
| 负迁移选择 baseline | PANDA reproduced | 当前核心 baseline |
| selector / routing 假设 | Reliability-aware Disagreement / Uncertainty-aware Stable-source / R3 Regret Router | 当前实现未通过稳定性或机制 gate，只作为 failed current implementations / evidence；全新 self-suppressed mixture 需新 D2/D3 -> D4 |
| Round 1 source-view/ranking/domain/reliability 假设 | R4 / P0-B / P1-A / P1-B / P1-C | 已按作用域关闭：R4/P0-B/P1-A 为 current D2 probe No-Go，P1-B Blocked，P1-C current D4+D5 No-Go；不进入正式主表 |
| Round 2 final-boundary/source-domain 候选池 | P2-A / P2-B / P2-C / P2-D | 已完成；无 `Primary-Candidate`，仅 P2-B final+aux residual 为 `Feasible-B` 弱信号；D3 结论只覆盖当前 offline/frozen 变体 |
| 当前新候选池 | R3-A / R3-B / R3-C / R3-D | 先做 train/val-only 最小 gate 与强 controls，过线后才进入三 seed val 复核 |

不建议作为主表可复现 baseline：

- RAMM：可作为 reported SOTA 或 related work，但当前代码可用性和正式录用状态仍需复核。
- 纯 LLM 判别方法：成本高、稳定性差、复现难，容易偏离本项目主线。

## Related Work 写作建议结构

1. Multi-modal Fake News Detection。
2. Cross-modal Consistency and Ambiguity Modeling。
3. Multi-domain Fake News Detection。
4. Negative Transfer and Neighbor-Domain Adaptation。
5. Reliability and Calibration in Fake News Detection。
6. Retrieval/Knowledge/LLM-enhanced Fake News Detection。

写作重点：

- 不把所有方法平铺成流水账。
- 明确指出 PANDA 相比 MMDFND/DAMMFND 的核心改进是显式选择邻域以缓解负迁移。
- 再指出 PANDA 仍存在可靠性、不确定性、高置信错判和 Gumbel selector 稳定性问题；本项目验证了若干 selector / routing 改进假设，但没有形成跨 seed 稳定性能闭环。
- 写作时避免因果过度表述：当前不能强写“图文冲突解释错误”，也不能把 clean multimodal disagreement / uncertainty-aware stable-source selection / R3 写成已成功新方法；R4 也必须等 gate 通过后才能升级为主方法。
