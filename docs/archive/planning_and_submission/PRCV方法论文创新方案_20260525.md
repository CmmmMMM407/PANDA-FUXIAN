## 已有证据对方法设计的约束

### 不能再主打的路线

1. 强 CLIP 图文冲突因果线不能主打。
   - CLIP-only val error AUC：Weibo `0.5087`，Weibo-21 `0.5314`。
   - Confidence-uncertainty val error AUC：Weibo `0.8589`，Weibo-21 `0.8768`。
   - 结论：图文语义冲突不是当前证据里的主解释变量。

2. Clean branch/fusion Reliability-aware Disagreement selector 不能作为正式方法。
   - Selector v2 5-epoch Go/No-Go 中 clean branch/fusion variants 没有超过 deterministic。
   - 改变 neighbor set 不等于改变 final label。

3. Uncertainty-aware stable-source winning control 不能升级为正式方法。
   - Seed42 小幅过线，但 seeds 2024/2026 复核后不稳定。
   - 三 seed val F1/Acc mean 低于 deterministic。
   - Seed2024 是关键反例。

4. Deterministic selector 可以作为强对照，但不能包装成创新。
   - Repeated-forward 显示 deterministic 能把 y_score 方差降到浮点噪声级。
   - 但这只是关闭 Gumbel 随机性的预期效果。

### 必须利用的正信号

1. PANDA 是 strongest reproduced baseline。
   - Weibo-21 Macro-F1/Acc/AUC：`0.9474 / 0.9474 / 0.9879`。
   - Weibo Macro-F1/Acc/AUC：`0.9415 / 0.9415 / 0.9866`。
   - PANDA 相比 DAMMFND 未达到显著优势，但总体最强。

2. PANDA 可靠性指标也最强。
   - Weibo-21 ECE/Brier/HCE：`0.0255 / 0.0418 / 0.0233`。
   - Weibo ECE/Brier/HCE：`0.0211 / 0.0440 / 0.0196`。

3. 不确定性/可靠性确实解释错误。
   - Confidence-uncertainty test error AUC：Weibo `0.8933`，Weibo-21 `0.9013`。
   - Fusion uncertainty 和 branch disagreement 也有诊断价值。

4. 路由边界非常脆弱。
   - 原 top1-top2 margin 不是关键边界。
   - 真正该正则的是 `topS - top(S+1)`。

5. R3 gate 已证明 self-domain route collapse 是硬失败点。
   - `effective_source_num_mean≈1.000006`，`self_top1_ratio=1.0`。
   - harmful / beneficial non-self alpha 约 `1e-7`。
   - 所有正向 R3 variants 未过 deterministic。
   - 所以当前不再推进 R3 v0/v1，优先尝试无需 self-vs-source router 的 R4 non-self source-domain intervention views。

6. R4 必须利用的设计约束。
   - self domain 必须从 intervention view 集合中 mask 掉。
   - non-self view 必须直接接 final-logit CE / consistency loss，不能被近零 route alpha 吃掉。
   - 第一版 MVP 不依赖 reliability weighting；reliability 只做二阶段扩展或消融。
   - test 只做 confirmatory，不能用于选择 view source、loss 权重或方法路线。
   - 训练前必须先证明 forced non-self view 是有意义干预，而不是普通多 forward。

## 方案 0：R4-PANDA，Non-self Source-domain Intervention Consistency

### 核心命题

```text
Instead of asking the model to learn whether to route to self or source domains,
explicitly expose it to non-self source-domain interventions and train the final classifier to remain correct and stable.
```

### 方法模块

1. Non-self Source-domain Intervention View Generator
   - 对每个样本取 PAD top-k non-self source domains。
   - self domain 被显式 mask，不参与 view 集合。
   - 生成 anchor logits `z_i^0` 和 forced non-self view logits `z_i^s`。

2. Intervention View CE
   - 每个 non-self view 直接预测原样本标签：

```text
L_view = mean_i mean_{s in V_i} CE(y_i, z_i^s)
```

3. Anchor-view Consistency
   - 让 non-self view prediction 与 deterministic anchor 保持一致：

```text
L_cons = mean_i mean_{s in V_i} JS(stopgrad(p_i^0), p_i^s)
```

4. Worst-view Robustness（二阶段）
   - 抑制最有害的 source intervention：

```text
L_worst = mean_i max_{s in V_i} CE(y_i, z_i^s)
```

5. Intervention Sensitivity Diagnostics
   - 报告 view 间 score std、anchor-view JSD、view flip rate、view correct rate 和 view loss by domain。

第一版 primary MVP 损失：

```text
L = L_anchor
  + lambda_view * L_view
  + lambda_cons * L_cons
```

预注册 primary config：

```text
K = 2
view_source = pad_top2_nonself
lambda_view = 0.1
lambda_cons = 0.05
anchor_detach = true
warmup_epoch = 1
consistency_mask = train-label CE/confidence mask only
```

### Frozen-view Validity Gate

第一步不训练，只用 reproduced deterministic checkpoint 导出 anchor、self-view、PAD top2 non-self、PAD bottom2 non-self、shuffled PAD 和 random non-self 3 seeds 的 view scores / CE / JSD / score std / flip rate / correct rate。若 forced views 几乎不改变 logits，或 PAD top2 不优于 random / bottom2，则 R4 training No-Go。

### 第一轮 Training Gate

只跑 Weibo-21 seed42 5-epoch。

必跑对照：

- `panda_gumbel_short5`
- `deterministic_eval_l0`
- `deterministic_train_l0`
- `old_winning_control`
- `duplicate_anchor_ce_control`
- `r4_self_view_control`
- `r4_view_ce_bottom2_nonself`
- `r4_shuffled_pad_rank_view_ce_cons`
- `r4_random_nonself_view_ce_cons_seed1/2/3`

R4 主候选：

- `r4_view_ce_top2_nonself`
- `r4_cons_only_top2_nonself`
- `r4_view_ce_cons_top2_nonself`

进入 seed 复核的最低线：

- frozen-view validity gate 已通过。
- val Macro-F1 和 Acc 都不低于 deterministic。
- 至少一个主指标提升 >= 0.3pp；低于 0.3pp 只算边缘。
- intervention sensitivity / JSD 下降 >= 10%，且 duplicate-anchor、self、random、bottom2、shuffled controls 不能同样达到。
- 主候选优于 `self_view_control`，否则只是重复 self 增强。
- 主候选优于 random non-self 3 seeds 的均值，否则不能声称 PAD top2 有意义。

若 seed42 未过线，按 D-level 止损：若只到 frozen/probe/offline gate，只能判当前 R4 变体 No-Go；若训练 smoke 未过，也只能判当前训练实现 No-Go。不复核 seeds 2024/2026，不扩 Weibo；全新 non-self/source protocol 需新 manifest + D4。

## 方案 1：R3-PANDA，Regret-Regularized Reliable Routing

状态：历史 R3 v0 routing 当前实现失败路线。R3 MVP 已完成 Weibo-21 seed42 5-epoch gate，所有正向 variants 未过 deterministic，并且出现完全 self-domain route collapse。以下内容仅保留为方法设计反思，不再作为当前执行路线；全新 self-suppressed / differentiable non-self source mixture 需新登记和 D2/D3 -> D4。

### 核心命题

```text
Domain prototype similarity should be treated as a prior, not as the final criterion of transfer.
The final criterion should be whether a source reduces supervised risk for the current sample.
```

### 二次自我反驳后的 v1 改造

#### 反驳 1：如果 self 参与 source softmax，R3 会自我塌缩

旧 v0：

```text
alpha_i,s = softmax(PAD_i,s + router_delta_i,s), s in all domains
```

问题：

- self domain 的 PAD prior 往往最大。
- source adapter 初始没有学到有效差异。
- `L_neg` 惩罚有害 source，却不主动奖励探索。
- 于是 router 最优捷径是永远选 self。

v1 修正：

```text
z_self_i = target-only anchor prediction
alpha_i,s = softmax(mask_self(PAD_i,s + router_delta_i,s)), s != target_domain
rho_i = sigmoid(g_i)
z_ext_i = routed external-source prediction
z_i = (1 - rho_i) * z_self_i + rho_i * z_ext_i
```

含义：

- self 不再作为 source 被路由选择。
- `rho_i` 单独决定是否使用外部迁移。
- external router 必须在外部 source 中学习选择，而不是用 self 逃避迁移。

#### 反驳 2：只惩罚正 regret 不足以学习 source utility

旧 v0：

```text
L_neg = sum_s alpha_i,s * relu(stopgrad(CE_s - CE_self) + margin)
```

问题：

- 它只告诉 router “别选坏 source”，不告诉它 “哪个 source 更好”。
- 当所有外部 source 初期都看起来坏时，`rho_i` 可能学成永远 0。

v1 增加 utility ranking：

```text
utility_i,s = CE_self_i - CE_s_i
q_i = softmax(stopgrad(utility_i / tau_u))
L_rank = KL(stopgrad(q_i) || alpha_i)
```

保留 `L_neg`，但把它从唯一目标降级为防负迁移约束：

```text
L_route = lambda_rank * L_rank + lambda_neg * L_neg + lambda_bal * L_balance
```

#### 反驳 3：candidate source logits 自己也需要训练

若 candidate logits 只通过 routed mixture 间接训练，低 alpha source 的 adapter 可能长期学不到东西，导致 regret target 噪声大。

v1 增加轻量 candidate auxiliary loss：

```text
L_cand = mean_{s != self} w_i,s * CE(y_i, z_i,s)
```

权重建议：

- warmup 阶段用 uniform weight，让每个 source adapter 至少学会基本分类。
- routing 阶段用 `detach(q_i)` 或 top-k utility weight，避免所有 source 被强行学成同质分类器。

#### 反驳 4：transfer gate 可能学成永远不用外部 source

v1 对 `rho_i` 加弱约束，只在 warmup 后使用：

```text
L_rho = max(0, rho_min - mean_batch(rho)) + max(0, mean_batch(rho) - rho_max)
```

建议范围：

```text
rho_min = 0.1
rho_max = 0.7
```

这不是强迫每个样本迁移，而是防止 batch 级完全不用或过度使用外部 source。

### 方法模块

1. Soft Source Utility Router
   - 输入：PAD logits、sample representation `h_i`、target/source domain embedding、detached reliability vector。
   - 输出：对非 self candidate source 的 soft route 权重 `alpha_i,s`。
   - 目的：替代 Gumbel top-S，让 source selection 可微。

2. Source Adapter Residual
   - 每个 source domain 配一个轻量 adapter。
   - 让 route 权重直接影响 final classifier input，而不是只影响 neighbor prompt。

3. Candidate Source Logits
   - 为每个 source 计算候选预测 `z_i,s`。
   - 用同一个 final head 或共享结构，保证 candidate utility 和主预测同构。

4. Negative Transfer Regret Loss
   - 定义：

```text
regret_i,s = CE(y_i, z_i,s) - CE(y_i, z_i,self)
L_neg = mean_i sum_s alpha_i,s * relu(stopgrad(regret_i,s) + margin)
```

   - 语义：若某 source 让训练损失高于 target/self anchor，它就是当前样本的潜在负迁移源。
   - 关键：只用 train label，不用 val/test error label。

5. Utility Ranking Distillation
   - 用 candidate source 相对 self anchor 的损失差构造 utility ranking。
   - 让 router 学习完整 source utility 排序，而不只是避开正 regret。

6. Transfer Gate
   - 单独学习 `rho_i`，决定当前样本是否需要外部 source。
   - 报告 `rho_mean`、`rho_by_domain`、`rho_by_uncertainty_bin`、`external_top1_source`。

7. Route Balance and Anti-Collapse
   - `L_balance = KL(mean_batch(alpha) || Uniform)` 或 entropy lower-bound。
   - 报告 `external_effective_source_num`、`rho_mean`、`external_top1_source_frequency`、`alpha_mean_per_source`、`route_entropy_by_domain`。

### v1 训练时序

| Stage | Epoch | 开启模块 | 目的 |
| --- | --- | --- | --- |
| Warmup | 1 | target anchor + external candidates + uniform `L_cand` | 先让 candidate source logits 有可用监督信号 |
| Exploration | 2 | external soft router + high tau + balance + weak `rho` lower bound | 防止 self-collapse，观察外部 source utility |
| Regret routing | 3-5 | `L_rank + L_neg + L_balance + L_rho` | 学习 source utility，抑制负迁移 |
| Stabilization | 5 后 | tau 退火，保留 diagnostics | 形成可解释路由模式 |

第一轮 5-epoch gate 至少应包含 warmup/exploration/regret 三个阶段；否则 R3 很可能因为初始 source logits 噪声被误杀。

### Go/No-Go

进入三 seed 复核的最低线：

- Weibo-21 seed42 val Macro-F1/Acc 不低于 deterministic。
- AUC、ECE、Brier、HCE、weak-domain F1、route collapse 中至少 2 项优于 deterministic。
- `external_effective_source_num` 明显高于塌缩值，建议 `> 1.5`。
- `rho_mean` 不应接近 0 或 1，建议 `[0.1, 0.7]`。
- `rank_neg` 必须优于或至少不弱于 `adapter_cand_aux_l0`，否则 regret/utility 不是贡献。
- `rank_neg` 必须优于 `neg_only` 或 `rank_only` 中至少一个，证明组合有意义。
- `rank_neg_reliability` 必须优于 no-reliability / random / overconfidence，才能在题目里保留 `Reliable`。
- `shuffled_utility` 不能同样有效，否则 utility ranking 主张不成立。
- `no_candidate_aux` 若不退化，说明 candidate auxiliary 不是必要模块，论文里应删掉或降级为稳定训练技巧。

升级正式方法主线的最低线：

- Weibo-21 seeds 42/2024/2026 三 seed val mean 不低于 deterministic。
- 至少 2/3 seeds Macro-F1/Acc 不退化。
- Seed2024 不能重演旧 winning-control 反例。
- Test 只做 confirmatory，不根据 test 回头调 `lambda_neg`、`tau`、`adapter_dim`、reliability 配方。

如果 R3 过线，论文贡献可写成：

1. 指出 PANDA prototype similarity 的边界：相似 source 不一定有正 supervised utility。
2. 提出 regret-regularized source utility routing，把 source selection 从相似性 prior 推进到效用学习。
3. 通过 route/regret/reliability diagnostics 证明减少 negative transfer，尤其改善 weak-domain 或可靠性指标。

### 审稿人可能攻击点与预先修正

| 攻击点 | 风险 | 修正 |
| --- | --- | --- |
| 这只是 MoE routing | 新颖性不足 | 强调 expert 不是通用专家，而是 source-domain transfer intervention；核心监督是 source regret / utility ranking |
| 用 train label 算 regret 是泄漏 | 方法被误解为错误检测器 | 明确只用训练监督，不使用 val/test error；所有超参由 val 固定，test confirmatory |
| 外部 source route 会塌缩 | 方法无效 | mask self + transfer gate + candidate auxiliary + route diagnostics |
| 提升来自 adapter 参数更多 | 被认为参数量增益 | 加 adapter-only、parameter-matched MLP、lambda_neg=0 控制 |
| 提升来自 uncertainty control | 撞旧 winning control | 加 no-reliability、random reliability、overconfidence、confidence-only controls |
| 单 seed 过拟合 | 可信度不足 | seed42 仅 gate；三 seed val mean 和 2/3 seeds 不退化后才扩 Weibo |

## 方案 2：R3-DRO，Group-Robust Regret Routing

### 适用条件

R3 MVP 在 overall F1/Acc 非退化，但 weak-domain F1 或 HCE 仍不稳时启用。

### 方法

在 R3 主损失上加入轻量 group risk：

```text
L_group = Var_g(mean_{i in g} CE(y_i, z_i))
```

group 优先级：

1. domain
2. domain x label
3. domain x uncertainty bin
4. val-defined weak-domain vs non-weak-domain

如果 batch 内 domain 太稀疏，用 domain risk EMA。

### 风险

- Group loss 容易牺牲 overall。
- 若和 regret loss 同时开启，失败后难定位原因。
- 所以它只能作为 R3 MVP 过线后的二阶段增强，不进入第一轮 gate。

## 方案 3：Evidence-R3，Evidential Reliable Routing

### 适用条件

R3 能提升 F1/Acc，但 reliability 指标如 ECE/Brier/HCE 改善不足时启用。

### 方法

给 text/image/fusion 分支增加 evidential heads：

```text
e_m = softplus(W_m h_m)
alpha_m = e_m + 1
u_m = K / sum(alpha_m)
```

把 branch evidence、fusion uncertainty、evidence conflict 作为 router condition，而不是直接用概率置信度。

### 可写贡献

- 从 probability uncertainty 升级为 evidence uncertainty。
- 区分 “low confidence due to lack of evidence” 和 “modal disagreement”。

### 风险

- PANDA 参数量已经很大，evidential head 会增加训练不稳定。
- 如果没有 F1/Acc 增益，审稿人可能认为是可靠性指标装饰。
- 不建议进入第一轮。

## 方案 4：Counterfactual Source Utility Distillation

### 核心命题

不要只惩罚负迁移 source，还要把 source utility 排序蒸馏给 router。

更新：该方案已并入 R3 v1 的 `L_rank`，不再作为完全独立路线。它仍可作为消融维度存在：`rank_only`、`neg_only`、`rank_neg`。

### 方法

在训练阶段离线/在线计算候选 source 的 counterfactual utility：

```text
utility_i,s = CE(y_i, z_i,self) - CE(y_i, z_i,s)
```

构造 soft target：

```text
q_i = softmax(utility_i / tau_u)
L_rank = KL(stopgrad(q_i) || alpha_i)
```

与 R3 的 `L_neg` 区别：

- `L_neg` 只惩罚有害 source。
- `L_rank` 学习完整 source utility ranking。

### 控制实验

- shuffled utility ranking。
- label-free PAD ranking。
- uncertainty-only ranking。
- target-only anchor。

### 风险

- 计算量更大。
- 训练早期 source logits 未成熟，utility target 可能噪声大。
- 建议在 epoch 1 warmup 后再开。

## 方案 5：Two-Timescale Reliable Prototype Memory

### 核心命题

旧 reliability prototype 失败，可能因为它只统计 domain mean，没有区分短期 batch 状态和长期稳定 source utility。

### 方法

维护两个 memory：

1. Slow domain memory
   - EMA source utility、uncertainty、ECE proxy、route success rate。
2. Fast sample-conditioned router
   - 当前样本根据 `h_i + reliability_i + PAD_i` 查询 memory。

路由分数：

```text
score_i,s = pad_i,s + mlp([h_i, r_i, slow_memory_s]) - beta * negative_regret_ema_s
```

### 风险

- 复杂度高，容易被认为工程堆叠。
- 需要非常清楚的 ablation。
- 适合作为 R3 成功后的增强，不作为主候选。

## 对实验思路的记录

1. 当前最有价值的思想不是“找到更好的相似度”，而是把 source domain 看成一种可造成负迁移的干预，必须用监督效用验证。
2. Reliability signal 应当是 router condition 或 robust training weight，而不是直接拿来重排 PAD logits。
3. 方法成败要看 final boundary 是否变化；neighbor set 变化只是中间现象。
4. Route collapse 是 R3 第一大风险，必须和 F1/Acc 同级记录。
5. 若 reliability vector 无贡献，题目中不能写 `Reliable`，应改成 `Regret-Regularized Source Utility Routing`。
6. 若 regret 无贡献但 adapter 有贡献，说明真正有效的是 source-specific residual，而不是 negative transfer reasoning；论文主张必须相应收缩。
7. 任何单 seed 过线都不能直接进入投稿叙事，seed2024 反例必须作为内部警戒线。

## 第三轮自我反驳：方法是否过重

### 问题

R3 v1 曾计划同时包含 external router、adapter、candidate logits、transfer gate、ranking、regret、balance、rho constraint。审稿人可能认为这是多模块堆叠；实验上也可能出现“过线了但不知道谁有用”的问题。由于 R3 v0 已出现 self-route collapse，当前不再把 R3 v1 作为执行主线。

### 修正原则

论文主方法只保留 3 个不可缺模块：

1. External source utility router
2. Transfer gate
3. Utility/regret supervision

以下模块只能作为训练稳定技巧或附录：

- candidate auxiliary loss
- route balance
- rho range constraint
- reliability vector

### 主文公式建议

主文只写：

```text
z_i = (1 - rho_i) z_self_i + rho_i sum_{s != d_i} alpha_i,s z_i,s
L_utility = KL(stopgrad(q_i) || alpha_i) + lambda_neg sum_s alpha_i,s relu(stopgrad(regret_i,s))
L = L_cls + L_aux + lambda_u L_utility
```

附录再写：

```text
L_stab = lambda_cand L_cand + lambda_bal L_balance + lambda_rho L_rho
```

这样主贡献清楚：不是“加很多正则”，而是“把 source selection 从 prototype similarity 改为 supervised utility routing”。

## 第四轮自我反驳：是否真的需要 reliability

当前证据显示 uncertainty/reliability 能解释错误，但旧 selector 没证明它能稳定提升。R3 的核心其实是 utility/regret，不是 reliability。如果 reliability vector 在 v1 中没有超过 no-reliability / random / overconfidence，论文题目必须删掉 `Reliable`。

命名分岔：

| 结果 | 标题 |
| --- | --- |
| reliability 有独立贡献 | Regret-Regularized Reliable Neighbor Adaptation |
| reliability 无贡献但 regret 有贡献 | Regret-Regularized Source Utility Routing |
| regret 无贡献但 adapter/gate 有贡献 | Source-Gated Neighbor Adaptation |
| 都无贡献 | 不写方法；只保留 diagnostic/report 作为 fallback |

主张顺序也应调整：

1. 第一贡献永远是 source utility / regret。
2. Reliability 只能作为 conditional routing feature，不作为第一贡献。
3. 诊断实验用于解释为什么需要从 similarity 转向 utility，而不是直接证明 reliability 方法有效。

## 第五轮自我反驳：PRCV 审稿风险清单

| 风险 | 会导致的审稿意见 | 必须补的证据 |
| --- | --- | --- |
| 只在 Weibo-21 有效 | 泛化不足 | Weibo 至少不退化；若 Weibo 不过，主张改成 boundary-condition paper |
| 提升低于 0.3 点 | 增益太小 | 强机制图：source utility heatmap、weak-domain route shift、regret distribution |
| 参数更多导致提升 | 不公平 | parameter-matched adapter-only / MLP control |
| 训练标签定义 utility 太直接 | 近似 sample reweighting | 说明 utility 是 source-level counterfactual，不是 error detector；加 shuffled utility control |
| test 参与方法选择嫌疑 | 实验不可信 | 全部 gate 使用 val；test 只 confirmatory；保存 config |
| 旧 PANDA 已 AAAI 2026，增量不足 | novelty 弱 | 强调从 prototype similarity 到 supervised source utility 的问题重定义 |

## 最终执行建议

当前不要继续跑 R3 v0 / v1，也不要把 R3 的 shuffled-regret 或 old winning control 包装成方法成功。

更优执行顺序：

1. 先在远端活代码仓库实现 R4 held-checkpoint forced-view export。
2. 只导出 train/val，不碰 test。
3. 先跑 frozen-view validity gate，确认 PAD top2 non-self intervention view 不是数值噪声，也不是 random/bottom2 同等有效。
4. validity 通过后再冻结 `r4_gate_config.yaml`，实现最小 `selector_mode=r4_intervention` 或等价训练开关。
5. 训练前做 batch-size 2 smoke：旧 checkpoint strict load、view sources 不含 self、view logits shape 正确、loss finite、diagnostics 完整。
6. 再跑 Weibo-21 seed42 primary gate；不过 deterministic / 0.3pp / JSD 下降要求就立即止损。
7. 每个新 variant 的 checkpoint 只保留远端路径，不同步公开日志仓库。
