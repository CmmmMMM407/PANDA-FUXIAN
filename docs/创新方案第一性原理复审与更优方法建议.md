## 0. 总结先行

当前建议不是推翻前一版 `Evidence-Gated Final Boundary + Aux-to-Final Meta Weighting`，而是把它升级：

```text
从 heuristic risk gate
升级为 counterfactual utility-supervised evidence gate。
```

也就是说，原方案的问题不是方向错，而是证据门控的监督信号还不够硬。过去 R7/R8 的失败已经说明：`uncertainty / disagreement / low-margin / confidence` 这些信号能解释错误，但很容易被 shuffled risk、confidence-only、random control、DWA/GradNorm 打穿。因此下一步不能只问“哪个分支更不确定”，而要问：

```text
如果拿掉某个分支、某个 source、某个 aux 梯度，
当前样本的 supervised loss 会变好还是变坏？
```

最推荐的新主线：

```text
CUE-PANDA:
Counterfactual Utility-aligned Evidence PANDA

核心：
1. 用 train-only counterfactual branch ablation 估计每个分支的边际效用。
2. 用边际效用监督 evidence gate，而不是只用 confidence/risk heuristic。
3. 用同一套 utility 信号调度 aux-to-final 权重。
4. 可选加入 DGL-style gradient disentanglement，避免 final fusion loss 与 unimodal aux loss 互相干扰。
5. 保留 PAD/DCA 作为 domain prior；source utility 作为第二阶段，而不是第一步硬上。
```

如果 CUE-PANDA 过 D5，它比“调 aux weight”或“加一个 risk gate”更像正式论文贡献：它回答的是 PANDA 没有回答的第一性问题：**对当前样本而言，哪条 evidence 的因果/边际收益最大？**

## 1. 对当前方案的反审

当前主线：

```text
Evidence-Gated Final Boundary
+ Aux-to-Final Meta Weighting
+ PAD-as-prior Source Utility reserve
```

### 1.1 优点

| 维度 | 判断 |
| --- | --- |
| 贴合源码瓶颈 | 强。直接命中 `h_di = h_text + h_image + h_fusion` 等权相加 |
| 贴合实验规律 | 强。R8-B 表明 aux supervision 有真实训练动力学线索 |
| 避免旧坑 | 较强。没有继续把 late PAD/GNS/DCA 调参当主线 |
| 工程可落地 | 强。不需要重训大 backbone，不需要外部知识库 |
| 审稿叙事 | 中高。能说清 branch evidence 如何进入 final boundary |

这个方向比继续调 selector/prompt 更稳，因为它把创新切口放在 PANDA 最可能的瓶颈上：branch evidence 和 final boundary 的错位。

### 1.2 最大风险

| 风险 | 为什么危险 | 对应已有失败 |
| --- | --- | --- |
| risk gate 被认为是 heuristic | uncertainty/disagreement 本身不等于有用 evidence | R8-A 被 shuffled risk control 打穿 |
| aux meta weighting 被认为是通用 MTL trick | 只调 loss weight 容易输给 DWA/GradNorm | R8-B seed2026 被 DWA 打穿 |
| evidence gate 退化为 confidence-only | final margin/entropy 常和错误相关，但不一定提供新机制 | confidence-only controls 多次接近 primary |
| frozen/offline gate 上限不足 | offline residual/adapter 经常被 Platt 或 stacking 打穿 | Round3/R4/R6 多个 offline No-Go |
| final boundary 没被真实塑造 | 分支信号只更新 aux head 或 branch path | R7-D/R8-C/R8-D 梯度 sanity 暴露路径问题 |

结论：

```text
原方案可以保留，但不能只靠 risk vector 做主监督。
必须加入“边际效用”这个更硬的训练信号。
```

## 2. 为什么“边际效用”是更好的第一性变量

PANDA 的核心问题是 negative transfer；当前模块审计进一步说明，PANDA 还存在 evidence transfer 的同构问题：

```text
source 是否有用，不能由 prototype similarity 直接决定；
branch 是否有用，也不能由 confidence 或 uncertainty 直接决定。
```

更基础的变量应该是 utility：

```text
Utility(evidence | x)
= 使用这条 evidence 后，当前样本 supervised loss 的预期下降。
```

这可以分成两层：

1. **Branch Utility**：文本、图像、fusion 哪个分支对当前样本最有帮助？
2. **Source Utility**：哪个邻域 source domain 对当前样本最有帮助？

过去我们一直在用 proxy：

- confidence proxy
- uncertainty proxy
- branch disagreement proxy
- PAD similarity proxy
- selector margin proxy

这些 proxy 都可能相关，但没有回答反事实问题。新的 CUE-PANDA 要先验证：

```text
counterfactual branch/source utility 是否比这些 proxy 更接近真实收益。
```

这也能解释为什么它比当前方案更强：它不是再设计一个更复杂的 gate，而是改变 gate 的监督目标。

## 3. 更优主线：CUE-PANDA

### 3.1 核心思想

CUE-PANDA 的核心是：

```text
用反事实边际效用监督 evidence gate，
再让 evidence gate 同时控制 h_di 聚合和 aux-to-final 训练预算。
```

基本链路：

```text
h_text, h_image, h_fusion
  -> branch aux heads
  -> counterfactual branch utility targets, train only
  -> utility predictor / evidence gate
  -> h_di_star
  -> original PAD/DCA h_collab
  -> final classifier

training:
  L_final
  + L_rec
  + utility-weighted L_aux
  + optional DGL-style gradient disentanglement
```

### 3.2 Branch Counterfactual Utility

对一个训练样本 `x`，先用当前或 warm teacher PANDA 得到：

```text
y_full = f(h_text + h_image + h_fusion, h_collab)
```

然后构造 leave-one-branch-out counterfactual：

```text
y_minus_text   = f(h_image + h_fusion, h_collab)
y_minus_image  = f(h_text + h_fusion, h_collab)
y_minus_fusion = f(h_text + h_image, h_collab)
```

也可以用 learned null branch / branch dropout 代替直接置零，避免分布外 ablation。

定义分支边际效用：

```text
u_text   = CE(y_minus_text, y)   - CE(y_full, y)
u_image  = CE(y_minus_image, y)  - CE(y_full, y)
u_fusion = CE(y_minus_fusion, y) - CE(y_full, y)
```

直觉：

- 如果拿掉 text 后 loss 上升，说明 text 对当前样本有正效用。
- 如果拿掉 image 后 loss 下降，说明 image 当前可能是负 evidence。
- 训练时只用 train labels 计算 utility target；推理时只用 utility predictor，不用 label。

标准化：

```text
u_pos = relu(u)
q_target = normalize(u_pos + epsilon)
```

如果三个分支都没有正效用，则退回保守分布：

```text
q_target = [1/3, 1/3, 1/3]
```

### 3.3 Utility-supervised Evidence Gate

训练一个轻量 gate：

```text
q_pred = softmax(g([
  h_text, h_image, h_fusion,
  p_text, p_image, p_fusion,
  entropy_text/image/fusion,
  branch disagreement,
  fusion uncertainty,
  domain embedding,
  PAD selector margin
]))
```

监督目标：

```text
L_utility_gate = KL(stopgrad(q_target) || q_pred)
```

然后生成新的 `h_di`：

```text
h_di_star =
  q_text  * h_text
+ q_image * h_image
+ q_fusion* h_fusion
+ residual_adapter([h_text, h_image, h_fusion, q_pred])
```

保守版先去掉 residual：

```text
h_di_star = sum_m q_m * h_m
```

这样第一轮可以减少参数量争议。

### 3.4 Utility-weighted Aux-to-Final

当前 R8-B 的线索是 aux 有用，但 static aux 2.0 不够稳。CUE-PANDA 的处理方式：

```text
L_aux_cue =
  stopgrad(q_text)   * BCE(p_text, y)
+ stopgrad(q_image)  * BCE(p_image, y)
+ stopgrad(q_fusion) * BCE(p_fusion, y)
```

更强版本加入梯度对齐：

```text
a_m = relu(cos(grad_hm L_final, grad_hm L_aux_m))
w_m = stopgrad(q_m) * stopgrad(a_m)
L_aux_cue = sum_m w_m * L_aux_m
```

这和 DWA/GradNorm 的区别：

- DWA/GradNorm 是 task/global/batch-level balancing。
- CUE 是 sample-level、branch-level、counterfactual-utility supervised weighting。

所以论文贡献更像机制，而不是通用训练 trick。

### 3.5 Optional：DGL-style Gradient Disentanglement

DGL 的启发是：多模态 fusion loss 反传到单模态 encoder 时可能削弱单模态优化，而 unimodal loss 也可能干扰 fusion module。迁移到 PANDA：

```text
branch experts:
  mainly updated by utility-weighted aux losses

evidence gate + final classifier:
  updated by final loss and utility gate loss

fusion / shared components:
  only接受不冲突或投影后的梯度
```

最小工程版：

- `L_final` 更新 final head、evidence gate、fusion adapter。
- `L_aux_cue` 更新 branch experts 和 aux heads。
- 对共享张量使用 PCGrad/CAGrad 或 stop-gradient split 作为 controls。

注意：PANDA 的 BERT/MAE/CN-CLIP 本来 frozen，所以 DGL 不应照搬到 backbone；应作用于 branch experts、fusion experts、evidence gate 和 final head 的梯度边界。

## 4. CUE-PANDA 相比当前方案更强在哪里

| 维度 | 当前 Evidence-Gated/Aux-to-Final | CUE-PANDA 升级 |
| --- | --- | --- |
| Gate 监督信号 | uncertainty/disagreement/confidence proxy | counterfactual loss reduction |
| 对 R8-A 的防御 | 可能仍被 shuffled risk 打穿 | shuffled utility target 是显式强对照 |
| 对 R8-B 的解释 | aux 有用，但 why 不清楚 | 哪个分支 aux 对哪个样本有正效用 |
| 与 DWA/GradNorm 区分 | 可能被认为是 MTL reweighting | sample-level branch utility，不是 global task weight |
| 与 PANDA 的关系 | 改 final boundary | 补足 PANDA 未建模的 evidence utility |
| 论文叙事 | 中高 | 更强：从 wise source selection 到 wise evidence utility |

一句话：

```text
PANDA 从 blind transfer 到 wise source selection；
CUE-PANDA 从 blind branch aggregation 到 utility-aligned evidence selection。
```

这条叙事和 PANDA 原动机同构，但不重复 PANDA；它补的是 PANDA 解决方案内部的新缺口。

## 5. 备选方法审查

### 5.1 DGL-Aux PANDA

定位：低工程风险、可作为 CUE-PANDA 的简化版或强对照。

核心：

```text
decouple branch optimization and final fusion optimization
```

做法：

- aux loss 主要更新 branch experts。
- final loss 主要更新 evidence gate / final classifier。
- fusion module 不被 unimodal aux 直接拉偏。
- 用 PCGrad/CAGrad 处理不可避免的共享梯度冲突。

优点：

- 直接解释 R8-B static aux 和 DWA 现象。
- 代码改动比 CUE 小。
- 有 DGL 这类多模态优化文献支撑。

缺点：

- 如果只做 gradient surgery，容易被认为泛化 trick。
- 必须结合 PANDA 的 branch/final 结构，不能只包装 DGL。

建议状态：

- 作为 Round9-B 或 CUE-PANDA ablation/control。
- 如果 CUE D2 utility signal 不成立，DGL-Aux 可以成为降级主线。

### 5.2 PAD-prior Counterfactual Source Utility

定位：高创新、高风险，适合第二阶段。

核心：

```text
source utility, not PAD similarity
```

做法：

```text
U_source(s,t,x)
= CE(y_without_s, y) - CE(y_with_s, y)
```

用 forced source / source adapter proxy 估计 source utility，再训练 source utility predictor：

```text
source_logits =
  log(PAD_sim)
+ beta * utility_pred
- gamma * negative_transfer_risk
```

优点：

- 直接回应 PANDA 的 blind transfer / negative transfer。
- 论文张力很高。

风险：

- 旧 source/prompt/PAD 线负证据较多。
- forced source counterfactual 可能是分布外 intervention。
- 如果 source 仍只影响 late DCA prompt，很可能重复失败。

建议状态：

- 不作为 Round9 第一主线。
- 等 CUE-PANDA 的 branch utility D2/D3.5 证明可行后，再把同一套 utility 思路迁移到 source。

### 5.3 Domain-View Risk Memory

定位：reserve，高分析价值。

核心：

```text
train-only memory over domain x class x view x risk-bin
```

优点：

- 能解释 hard region。
- 可作为 evidence gate 的辅助 signal。

风险：

- 容易变成 kNN 后处理。
- 容易被 label-shuffled memory / class-prior memory 打穿。

建议状态：

- 先做 D2/D3 upper-bound，不急着训练。

### 5.4 Evidence Sidecar / RAG

定位：高风险未来线。

优点：

- 事实核验方向有高创新张力。
- 对“文本和图片本身不足以判断真假”的样本自然。

风险：

- temporal leakage、检索质量、复现成本、LLM hallucination。
- 与 PANDA baseline 边界不清。

建议状态：

- 不进当前 Round9 主线。
- 先做 extraction/retrieval audit，后续作为 future work 或第二论文方向。

## 6. Round9 方案重排

执行后状态（2026-05-29）：

- Round9-A CUE D2 已完成，结论 `No-Go-for-current-CUE-D2`。`cue_gate` 低于 confidence-only、same-param MLP、stacking/Platt controls；CUE flip audit 为 W2C `8`、C2W `29`。
- Oracle val utility gate diagnostic 达到 F1/Acc/AUC `0.973959/0.973984/0.998773`，说明 branch counterfactual utility 有强上界信号，但当前 train-only gate 泛化失败。
- Round9-C DGL-Aux D4 已完成，结论 `No-Go-for-current-DGL-Aux`。Best DGL 低于 static aux 2.0、DWA、deterministic/same-budget/detached controls，且 flip net 为负。

因此，Round9 后续不再推进 CUE D3.5/D4/D5；新的主线必须改变 utility 的使用方式：不再让 utility 直接控制 inference-time evidence aggregation，而是把它降级为 training-time auxiliary supervision allocator。

### P0：Round9-A CUE Branch Utility Gate

目标：

验证 branch counterfactual utility 是否比 confidence/uncertainty/disagreement 更适合作为 evidence gate 监督。

最小步骤：

1. 用现有 PANDA checkpoint 导出 train/val branch features、aux logits、final logits。
2. 做 leave-one-branch-out counterfactual forward。
3. 计算 train-only utility target。
4. 训练极小 utility predictor。
5. 在 val 上比较：
   - original equal sum
   - global branch weights
   - confidence-only gate
   - uncertainty/disagreement gate
   - CUE gate
   - random utility
   - shuffled utility
   - same-param MLP
   - final+aux stacking
   - Platt/temperature

Go 条件：

- CUE frozen/offline 至少优于 confidence-only、shuffled utility、same-param MLP。
- hard-bin wrong-to-correct 明显大于 correct-to-wrong。
- 不是只改善 ECE/Brier。

No-Go 条件：

- utility target 与 val 改善无关。
- shuffled utility 与真实 utility 持平。
- confidence-only gate 追平或更强。
- final+aux stacking/Platt 更强。

### P0：Round9-B CUE Training D3.5/D4

只有 Round9-A D2/D3 过线后启动。

D3.5：

- 检查 `h_di_star` 梯度非零。
- 检查 final classifier 梯度非零。
- 检查 high-utility branch 梯度富集高于 shuffled utility。
- 检查 aux-to-final gradient cosine 不被严重冲突破坏。

D4：

- seed42 train/val-only。
- controls 必须包括：
  - deterministic PANDA
  - static aux 2.0
  - DWA
  - GradNorm
  - PCGrad/CAGrad
  - confidence-only gate
  - shuffled utility gate
  - random utility gate
  - same-budget CE

D5：

- seeds 42/2024/2026 三 seed val。
- 只有三 seed 过线才升 Primary-Candidate。

### P1：Round9-C DGL-Aux PANDA

若 CUE utility D2 成立，DGL-Aux 作为 ablation/control。  
若 CUE utility D2 不成立，DGL-Aux 作为降级候选。

必须打过：

- static aux 2.0。
- DWA/GradNorm/PCGrad/CAGrad。
- detached aux。
- final-only same-budget。

如果只赢 deterministic，不赢 DWA，则只能写 auxiliary optimization diagnostic。

### P1：Round10 Source-CUE

启动条件：

- CUE branch utility 至少 D4 有正向趋势。
- 或 forced source utility D2 直接证明 source marginal utility 与 PAD 不同且优于 controls。

不满足则不启动，避免继续在 source/prompt 上空转。

## 6.5 Round10 新融合候选：BUA-PANDA

### 6.5.1 方法命名

```text
BUA-PANDA:
Boundary-gated Utility-Anchored Auxiliary PANDA
```

中文解释：

```text
边界门控的效用锚定辅助监督 PANDA
```

它整合三条已有线索：

- **R8-B static aux 2.0**：提供稳定训练锚点。已有三 seed 证据显示它稳定高于 deterministic，但 seed2026 被 DWA 打穿，说明“全局固定强度”还不够。
- **Round9-A oracle branch utility**：提供 branch-level 上界信号。当前 train-only CUE gate 泛化失败，但 oracle val utility 极强，说明 utility 不是无效变量。
- **R7-A/R8-A boundary-risk**：提供可靠性开关。risk 直接做训练目标会被 shuffled control 打穿，但它仍可用于判断“什么时候允许相信 utility 自适应”。

一句话主张：

```text
不是用 utility 改推理输出，而是用 utility 改训练时 aux supervision 的分支分配；
不是用 boundary-risk 当优化目标，而是用它决定是否允许偏离 static aux anchor。
```

### 6.5.2 第一性假设

PANDA 当前的 branch auxiliary heads 已经有训练价值，但默认 `mean(aux BCE)` 和 static aux 2.0 都是全局平均策略。更合理的问题不是“aux 要不要更强”，而是：

```text
对当前样本，哪个 branch 的辅助监督最可能塑造 final boundary？
这个自适应分配何时可靠，何时应该退回 static aux？
```

BUA-PANDA 的基本假设：

1. Static aux 2.0 是强 baseline，应作为 anchor，而不是被复杂模块替代。
2. Counterfactual utility 只能在训练期作为 branch aux 权重监督，不能直接做 inference gate。
3. Boundary-risk 只能做 trust gate，不能直接作为 loss reweight 主目标。
4. 任意自适应策略必须在不可靠时退回 uniform/static aux，避免重复 CUE gate 的泛化失败。

### 6.5.3 最小公式

训练损失：

```text
L =
  L_final
+ L_rec
+ lambda_i * sum_b w_i,b * L_aux_b
```

其中 `b in {text, image, fusion}`。

Aux strength 以 R8-B 为锚：

```text
lambda_i = clip(2.0 + delta_i, 1.5, 2.5)
```

第一版也可以固定：

```text
lambda_i = 2.0
```

Branch aux 权重：

```text
w_i,b = (1 - alpha_i) / 3 + alpha_i * q_i,b
```

`q_i,b` 来自 train-only counterfactual utility target：

```text
u_i,b = CE(y_minus_b, y_i) - CE(y_full, y_i)
q_i = normalize(relu(u_i) + epsilon)
```

`alpha_i` 是 trust gate：

```text
alpha_i =
  boundary_trust_i
* utility_confidence_i
```

推荐第一版实现：

```text
boundary_trust_i = 1[low_margin_i or high_boundary_risk_i]
utility_confidence_i = 1 - normalized_entropy(q_i)
alpha_i = clip(boundary_trust_i * utility_confidence_i, 0, alpha_max)
```

其中 `alpha_max` 建议先设为 `0.5`，保证不会完全偏离 static aux anchor。

关键退路：

```text
if utility unreliable or non-boundary sample:
  alpha_i = 0
  w_i = [1/3, 1/3, 1/3]
  method degenerates to static aux 2.0
```

这使 BUA-PANDA 和 Round9-C DGL-Aux 区分开：BUA 不做通用梯度手术，而是有约束地分配 branch aux supervision。

### 6.5.4 为什么它比三条旧线更合理

| 失败线索 | 失败原因 | BUA 的修正 |
| --- | --- | --- |
| R8-B static aux 2.0 | 全局强度有效但不稳，被 DWA seed2026 打穿 | 保留 2.0 anchor，只在可信边界样本上做局部偏移 |
| Round9-A CUE gate | train-only utility gate 做 inference aggregation 泛化失败 | utility 不再改推理输出，只改训练期 aux 分配 |
| R7/R8 boundary-risk | risk 直接做目标/权重会被 shuffled risk 打穿 | risk 只做 trust gate，且必须过 shuffled-risk ablation |
| Round9-C DGL-Aux | gradient surgery 不能超过 static aux/DWA | 不靠通用梯度手术，改成 sample-level branch aux allocation |

### 6.5.5 预注册 ablation

BUA-PANDA 必须至少包含以下 variants：

- `bua_anchor_static_aux2p0`：等价 static aux 2.0 anchor， sanity anchor。
- `bua_utility_only_aux_alloc`：只用 `q_i` 分配 aux，不用 boundary trust。
- `bua_boundary_only_aux_strength`：只用 boundary-risk 调 `lambda_i`，不使用 utility branch allocation。
- `bua_boundary_gated_utility_aux`：主方法，boundary trust + utility aux allocation。
- `bua_entropy_gated_utility_aux`：只用 utility entropy 控制 `alpha_i`，检验 boundary 是否必要。
- `bua_shuffled_utility_control`：打乱 utility target。
- `bua_shuffled_boundary_control`：打乱 boundary-risk/bin。
- `bua_random_utility_control`：随机 utility target。
- `bua_negative_utility_reverse_control`：反向使用 low-utility branch，检验是否只是噪声正则。
- `bua_oracle_val_utility_diagnostic`：只作上界诊断，不参与方法选择。

### 6.5.6 强对照 moat

进入 D4 时必须同时比较：

- deterministic PANDA。
- same-budget CE / noop。
- static aux 2.0。
- best static aux sweep if available。
- generic DWA。
- generic GradNorm。
- generic PCGrad / CAGrad。
- detached aux no-feature-update。
- utility-only without boundary。
- boundary-only without utility。
- shuffled/random/reverse utility。
- shuffled boundary-risk。

Go 条件：

- Seed42 D4 主方法 F1/Acc 同时高于 static aux 2.0、DWA、deterministic/same-budget/detached controls。
- Flip audit 净正：W2C > C2W。
- low-margin 或 weak-domain 至少一个子集提升，且 high-confidence correct 不显著被翻坏。
- shuffled utility / shuffled boundary / random utility 不能接近或超过主方法。
- ECE/Brier 改善不能作为唯一 Go 理由。

No-Go 条件：

- 主方法低于 static aux 2.0 或 DWA。
- utility-only 与主方法持平，说明 boundary gate 没有贡献。
- shuffled/random/reverse utility 与主方法持平，说明 utility signal 不干净。
- boundary-only 与主方法持平，说明 utility branch allocation 没有贡献。
- flip net 非正。

### 6.5.7 执行层级

BUA 不应直接开 D4。推荐从 D2.5 开始：

```text
D2.5 offline allocator simulation
  验证用 train-derived utility/boundary 产生的 aux allocation 是否和 val error/hard bin 有稳定相关。

D3.5 gradient sanity
  验证 BUA aux allocation 能触达 branch expert、h_di/final boundary，
  且 high-utility branch 梯度富集高于 shuffled/random controls。

D4 seed42 training smoke
  5-epoch train/val-only，完整强对照 moat。

D5 three-seed val
  仅 D4 过线后启动 seeds 42/2024/2026。
```

### 6.5.8 论文叙事边界

### 6.5.8a 执行后修正（2026-05-29）

Round10-A D2.5 已验证当前 BUA allocator：

- `bua_boundary_gated_utility_aux` expected utility `0.773082`，高于 static anchor `0.503536`、shuffled utility `0.539701`、random utility `0.508881`、reverse utility `0.468324`、confidence-only allocation `0.505511`。
- 但它低于 `bua_utility_only_aux_alloc` 的 `0.911898` 和 `bua_entropy_gated_utility_aux` 的 `0.816022`。
- 决策理由为 `boundary_gate_not_proven_vs:bua_utility_only_aux_alloc,bua_entropy_gated_utility_aux`。

因此本文原本的 BUA 假设需要修正：

```text
utility as branch-aux allocation signal: still promising;
low-margin/high-risk hard boundary trust gate: not supported in current form.
```

下一版如果重开，不应继续把 R7/R8 risk 作为 hard trust gate 续参；更值得探索的是 utility entropy / soft reliability calibration / train-time representation coupling。

若 BUA 过线，论文贡献可以写成：

```text
PANDA performs source-domain selection but still applies blind branch auxiliary supervision.
BUA-PANDA introduces boundary-gated counterfactual utility allocation,
which preserves the stable static auxiliary anchor while selectively emphasizing useful branch supervision near the final decision boundary.
```

若 BUA 只赢 deterministic、不赢 static aux 2.0/DWA，只能写成：

```text
auxiliary supervision sensitivity / training dynamics diagnostic
```

不能写成新主方法。

## 7. 最小可执行实验设计

### 7.1 D2 Counterfactual Utility Probe

输入：

- train/val split。
- existing PANDA checkpoint。
- branch features and logits。

输出：

- `branch_utility_train.csv`
- `branch_utility_val_diagnostic.csv`
- `cue_d2_summary.json`
- `cue_d2_decision.md`

核心统计：

- branch utility distribution by domain/class。
- top-utility branch vs branch correctness。
- utility entropy vs final error。
- utility target vs confidence-only correlation。
- true utility vs shuffled/random controls。
- val frozen gate Macro-F1/Acc/AUC/ECE/Brier/HCE。
- flip audit。

注意：

- train labels 可用于训练 utility predictor。
- val labels 可用于 gate 选择和决策。
- test 不导出、不读取、不分析。

### 7.2 D3.5 Gradient Sanity

必须记录：

- `h_di_star` gradient norm。
- final classifier gradient norm。
- branch experts gradient norm。
- `L_final` 与 `L_aux_cue` gradient cosine。
- high-utility over low-utility gradient ratio。
- true utility vs shuffled utility enrichment。

No-Go：

- final classifier 无梯度。
- CUE enrichment 不高于 shuffled/random。
- gradient cosine 系统性负且 PCGrad/CAGrad 无法修复。

### 7.3 D4/D5 Training

D4 seed42：

- 5 epoch 或当前 PANDA 对齐 budget。
- 保存 manifest、train curve、val metrics、flip audit、branch telemetry。
- 不保存 test、不保存大预测长表到公共同步。

D5 三 seed：

- 只在 D4 打过强 controls 后启动。
- D5 冻结 final config 前不允许看 test。

## 8. 论文叙事建议

### 8.1 题目方向

可选题名：

```text
Utility-Aligned Evidence Routing for Multi-Domain Multimodal Fake News Detection
```

或：

```text
From Wise Source Selection to Utility-Aligned Evidence Selection in Multi-Domain Fake News Detection
```

### 8.2 贡献写法

贡献 1：

```text
We identify a hidden bottleneck in PANDA-style neighbor-domain adaptation:
domain/source selection is modeled explicitly, but multimodal branch evidence is still aggregated blindly.
```

贡献 2：

```text
We propose counterfactual branch utility as a supervised training signal for evidence routing.
```

贡献 3：

```text
We align auxiliary branch supervision with final-boundary shaping through utility-weighted aux training and gradient diagnostics.
```

贡献 4：

```text
We provide strict controls showing the gains are not from confidence heuristics, random perturbation, calibration, or generic multitask balancing.
```

### 8.3 与 PANDA 的关系

不要写：

```text
PANDA wrong, our selector better.
```

建议写：

```text
PANDA solves blind source transfer at the domain level.
However, its internal multimodal evidence aggregation remains source-agnostic and sample-agnostic.
We complement PANDA by estimating utility at the evidence level.
```

这样既尊重 baseline，又有清晰突破口。

## 9. 反审问题预案

### Q1：这是不是只是 attention/gating？

回答要点：

- 普通 attention/gating 没有 supervised utility target。
- CUE 的 gate target 来自 branch removal 的 counterfactual loss change。
- 我们会报告 random/shuffled utility、confidence-only gate、same-param MLP controls。

### Q2：这是不是只是在调 aux loss？

回答要点：

- static aux 2.0、DWA、GradNorm、PCGrad/CAGrad 都作为强 controls。
- CUE 的 aux 权重是 sample-level branch utility，而不是 epoch-level task weighting。
- D3.5 会证明 aux signal 触达 final boundary。

### Q3：counterfactual ablation 是否分布外？

回答要点：

- 第一版会同时测试 zero ablation、learned-null ablation、branch dropout ablation。
- 如果三者结论不一致，只保留最稳定且 control moat 最强的版本。
- ablation target 只作为训练监督，不作为推理 oracle。

### Q4：是否使用了 val/test label 泄漏？

回答要点：

- utility predictor 只用 train labels 训练。
- val labels 只用于方法选择，与当前所有 train/val-only gate 口径一致。
- test 在 D5 final config freeze 后一次性使用。

### Q5：为什么不用 RAG/LLM 证据？

回答要点：

- 当前目标是先修复 PANDA 内部 evidence aggregation 和 optimization bottleneck。
- RAG/LLM 有 temporal leakage 和复现风险，更适合作为后续 sidecar。
- CUE 不依赖外部知识，更适合先形成稳健主线。

## 10. 最终建议

当前最值得推进的不是“再想一个全新大模块”，而是：

```text
把 Evidence-Gated PANDA 升级成 CUE-PANDA：
用 counterfactual branch utility 监督 evidence gate，
用同一 utility 信号组织 aux-to-final training。
```

具体执行顺序：

1. 先做 `Round9-A CUE D2 Counterfactual Utility Probe`。
2. 若 CUE utility 打不过 confidence/shuffled/stacking controls，立即关闭 CUE 当前变体。
3. 若 CUE D2 过线，做 D3.5 gradient sanity。
4. D3.5 过线后才做 D4 seed42 training smoke。
5. D4 打过 static aux 2.0、DWA、GradNorm、PCGrad/CAGrad 后，才做 D5 三 seed。
6. 若 CUE 失败，降级验证 DGL-Aux PANDA；若 DGL 也失败，停止方法赛马，转诊断型论文叙事。

一句话：

```text
PANDA 选择 domain；
CUE-PANDA 选择 evidence；
Aux-to-Final 让被选择的 evidence 真正塑造 final boundary。
```
